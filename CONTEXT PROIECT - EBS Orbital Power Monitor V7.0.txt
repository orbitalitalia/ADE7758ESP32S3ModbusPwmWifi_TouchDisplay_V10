📁 CONTEXT PROIECT - EBS Orbital Power Monitor V7.0
📋 INFORMAȚII GENERALE
Nume proiect: ESP32-S3 Power Monitor cu ADE7758
Versiune firmware: V7.0 (test OTA V8.1)
Hardware:

ESP32-S3 Dev Module (16MB Flash, 8MB PSRAM)

ADE7758 Energy Meter (SPI: CLK=8, MISO=6, MOSI=7, CS=5)

OLED SSD1306 (I2C: SDA=48, SCL=47)

DAC MCP4725 (I2C addr 0x60)

Ethernet W5500 (SPI: MISO=12, MOSI=11, SCLK=13, CS=14, RST=9, IRQ=10)

EEPROM I2C (24C256, addr 0x50) pentru NVRAM

IP-uri:

WiFi AP: 192.168.1.177 (întotdeauna activ)

Ethernet: 192.168.1.20 (când cablu conectat)

Modbus Slave: 192.168.1.246 (port 502)

✅ CE FUNCȚIONEAZĂ DEJA
Măsurători electrice (ADE7758)
Citire VRMS (tensiune) pentru fazele A, B, C

Citire IRMS (curent) pentru fazele A, B, C

Citire putere activă (Watt) per fază și total

Citire putere aparentă (VA) per fază și total

Calcul putere reactivă (Var) per fază și total

Calcul factor putere (PF) corect folosind delta WATTHR/VAHR

Citire frecvență (FREQ) cu calibrare (FREQ_CAL=16.14)

Filtrare EMA pentru PF (alpha=0.3) pentru stabilitate

Control disipare
PWM pe GPIO21 (canal 0, 3kHz, rezoluție 12 biți)

Step2 pe GPIO16, Step3 pe GPIO17, Step4 pe GPIO18

Alimentare generală stepuri pe GPIO38 (LOW=ON, HIGH=OFF)

Logică automată bazată pe putere măsurată vs setpoint

Control manual prin WebUI/WebSocket

Network & Comunicare
WiFi Access Point permanent (SSID: EBS-Orbital-Config, pass: 12345678)

Ethernet W5500 cu IP static 192.168.1.20

WebServer pe port 80 (accesibil prin ambele interfețe)

WebSocket pe port 81 pentru update în timp real

Modbus TCP client către 192.168.1.246:502

Scriere registre: WATT (reg 2017), STAY_ALIVE (reg 2021)

Citire Input Register 13 (Potenza rete) - PowerRete în kW

Persistență date (NVRAM)
Salvare setări în EEPROM I2C (24C256)

Factori calibrare: voltageFactor, currentFactor

Praguri disipare: prez1, prez2, prez3, prez4 (int)

Setpoint (W)

Energie acumulată (KWh)

CT Ratio: CT_Primary, CT_Secondary

OFFSET-URI în COUNTS: voltageOffsetA/B/C, currentOffsetA/B/C

Flag factoryCalDone

IP Modbus slave (4 bytes)

Stare alimentare stepuri (stepsEnabled)

Interfață utilizator
OLED Display (actualizare la 200ms)

Touch pe GPIO4 pentru control display (timeout 60 sec)

WebUI complet funcțional cu toate comenzile

Auto-Zero calibration din WebUI

OTA Update cu autentificare (admin/orbital123)

Buton restart ESP32 în WebUI

OTA & Rollback
Configurare rollback activată în sdkconfig

Două partiții OTA (OTA_0 și OTA_1)

Confirmare firmware după 15 sec în loop()

Verificare stare partiție la boot

Endpoint /update pentru upload firmware

🐛 PROBLEME CUNOSCUTE / ÎN LUCRU
Probleme active
🔴 Modbus TCP reconectare - La căderea cablului Ethernet, sistemul detectează corect, dar reconectarea uneori eșuează

🟡 Calibrare offset - Auto-zero din WebUI funcționează, dar offset-urile sunt în VOLȚI/AMPERI în UI, în timp ce în cod sunt în COUNTS - conversia trebuie verificată

🟡 Putere reactivă - Calculată ca sqrt(S² - P²), dar nu este verificată cu aparatură de referință

🔴 OTA cu semnătură - Codul pentru verificare RSA există dar e comentat (dev mode)

Probleme rezolvate recent
SPI timing pentru ADE7758 - delay-uri optimizate

Calcul PF corect (deltaWATTHR/deltaVAHR)

Safe GPIO init la boot (toate ieșirile LOW)

Timeout mod manual (60 sec) - reset automat

Citire PowerRete prin Modbus Input Register 13
